# 小程序运营闭环检查清单

## 生产环境配置

- 后端必须配置生产数据库 `DATABASE_URL`，并完成 Prisma migration deploy。
- 后端必须配置 `JWT_ACCESS_SECRET` 与 `JWT_REFRESH_SECRET`，不要复用本地默认值。
- 小程序真实微信登录必须配置 `WECHAT_APPID` 与 `WECHAT_SECRET`。
- 小程序端 `API_BASE_URL` 必须指向 HTTPS API 域名。
- 微信公众平台必须配置合法请求域名，且域名证书有效。
- 部署后验证 `/api/health` 与 `/api/health/db` 均返回成功。

## 运营 smoke 流程

- 管理员登录后台。
- 小程序用户提交续费申请。
- 后台财务页点击「待处理续费」，确认出现 `MEMBERSHIP_RENEWAL + PENDING` 交易。
- 将续费交易从「待处理」推进到「处理中」，再推进到「已完成」。
- 小程序用户提交意见反馈。
- 后台通知页点击「小程序反馈」，确认出现 `MINI_PROGRAM_FEEDBACK + INTERNAL` 通知。
- 将反馈标记为已处理，并确认状态变为已读。

## 后续增强

- 待处理续费列表当前一次加载 100 条；如果运营量增大，应改为分页列表或独立续费工作台。
- 反馈当前复用通知已读状态表达「已处理」；如果需要客服协作，应新增反馈工单状态、负责人、处理备注与关闭原因。
- 续费完成目前只更新交易状态；如果后续需要自动延长会员权益，应把交易完成与会员卡有效期/次数变更做成受控服务流程。
